## Gesture handlers

Motion Start provides gesture recognition out of the box, turning pointer events into meaningful interactions. The gesture system is only available in the full `Motion`/`motion` variant (not in `M`/`m`).

```typescript
interface MotionProps extends
  PanHandlers,
  TapHandlers,
  HoverHandlers,
  FocusHandlers,
  DraggableProps
{
  // ... other props
}
```

<Note>
For gesture features, make sure to import from `motion` or `Motion`, not the minimal `m` or `M` variants.
</Note>

## Tap gestures

Detect tap/click interactions with position information:

```typescript
export interface TapInfo {
  /**
   * Contains x and y values for the tap gesture relative to the
   * device or page
   */
  point: Point2D
}

export interface TapHandlers {
  /**
   * Callback when the tap gesture successfully ends on this element
   */
  onTap?(event: MouseEvent | TouchEvent | PointerEvent, info: TapInfo): void
  
  /**
   * Callback when the tap gesture starts on this element
   */
  onTapStart?(event: MouseEvent | TouchEvent | PointerEvent, info: TapInfo): void
  
  /**
   * Callback when the tap gesture ends outside this element
   */
  onTapCancel?(event: MouseEvent | TouchEvent | PointerEvent, info: TapInfo): void
  
  /**
   * Properties or variant label to animate to while the component is pressed
   */
  whileTap?: VariantLabels | TargetAndTransition
}
```

```svelte
<script>
  import { Motion } from 'motion-start'
  
  function handleTap(event, info) {
    console.log('Tapped at:', info.point.x, info.point.y)
  }
</script>

<Motion.button
  onTap={handleTap}
  whileTap={{ scale: 0.95 }}
>
  Tap me
</Motion.button>
```

## Hover gestures

Respond to pointer hover events:

```typescript
export interface HoverHandlers {
  /**
   * Properties or variant label to animate to while the hover gesture is recognised
   */
  whileHover?: VariantLabels | TargetAndTransition
  
  /**
   * Callback function that fires when pointer starts hovering over the component
   */
  onHoverStart?(event: MouseEvent, info: EventInfo): void
  
  /**
   * Callback function that fires when pointer stops hovering over the component
   */
  onHoverEnd?(event: MouseEvent, info: EventInfo): void
}
```

```svelte
<script>
  import { Motion } from 'motion-start'
</script>

<Motion.div
  whileHover={{ 
    scale: 1.05,
    transition: { duration: 0.2 }
  }}
  onHoverStart={() => console.log('Hover start')}
  onHoverEnd={() => console.log('Hover end')}
>
  Hover over me
</Motion.div>
```

<Info>
Hover gestures only work with mouse input. Touch devices don't support hover events.
</Info>

## Pan gestures

Track continuous pointer movement with velocity and offset information:

```typescript
export interface PanInfo {
  /**
   * Contains x and y values for the current pan position relative
   * to the device or page
   */
  point: Point2D
  
  /**
   * Contains x and y values for the distance moved since the last event
   */
  delta: Point2D
  
  /**
   * Contains x and y values for the distance moved from the first pan event
   */
  offset: Point2D
  
  /**
   * Contains x and y values for the current velocity of the pointer, in px/ms
   */
  velocity: Point2D
}

export interface PanHandlers {
  /**
   * Callback function that fires when the pan gesture is recognised on this element
   */
  onPan?(event: MouseEvent | TouchEvent | PointerEvent, info: PanInfo): void
  
  /**
   * Callback function that fires when the pan gesture begins on this element
   */
  onPanStart?(event: MouseEvent | TouchEvent | PointerEvent, info: PanInfo): void
  
  /**
   * Callback function that fires when we begin detecting a pan gesture.
   * This is analogous to onMouseStart or onTouchStart
   */
  onPanSessionStart?(event: MouseEvent | TouchEvent | PointerEvent, info: EventInfo): void
  
  /**
   * Callback function that fires when the pan gesture ends on this element
   */
  onPanEnd?(event: MouseEvent | TouchEvent | PointerEvent, info: PanInfo): void
}
```

```svelte
<script>
  import { Motion } from 'motion-start'
  
  function handlePan(event, info) {
    console.log('Position:', info.point)
    console.log('Movement:', info.delta)
    console.log('Total offset:', info.offset)
    console.log('Velocity:', info.velocity)
  }
</script>

<Motion.div
  onPan={handlePan}
  style={{ touchAction: 'none' }}
>
  Pan me
</Motion.div>
```

<Warning>
For pan gestures to work correctly with touch input, set `touch-action: none` CSS on the element to disable browser scrolling on that axis.
</Warning>

### Pan session events

Track the complete pan lifecycle:

```svelte
<script>
  import { Motion } from 'motion-start'
  
  let isPanning = false
  let totalDistance = 0
  
  function handlePanStart(event, info) {
    isPanning = true
    console.log('Pan started at:', info.point)
  }
  
  function handlePan(event, info) {
    totalDistance = Math.sqrt(
      info.offset.x ** 2 + info.offset.y ** 2
    )
  }
  
  function handlePanEnd(event, info) {
    isPanning = false
    console.log('Pan ended. Total distance:', totalDistance)
    console.log('Final velocity:', info.velocity)
  }
</script>

<Motion.div
  onPanStart={handlePanStart}
  onPan={handlePan}
  onPanEnd={handlePanEnd}
  style={{ 
    touchAction: 'none',
    backgroundColor: isPanning ? '#0099ff' : '#ccc'
  }}
>
  {isPanning ? `Panning: ${totalDistance.toFixed(0)}px` : 'Start panning'}
</Motion.div>
```

## Drag gestures

Enable drag interactions with constraints and momentum:

```typescript
export interface DraggableProps {
  /**
   * Enable dragging. true = both axes, 'x' or 'y' = single axis
   */
  drag?: boolean | 'x' | 'y'
  
  /**
   * If true, component can be dragged beyond constraints and will
   * snap back with spring animation
   */
  dragElastic?: DragElastic
  
  /**
   * Constraints on draggable area
   */
  dragConstraints?: false | Partial<BoundingBox2D> | RefObject<Element>
  
  /**
   * Applies momentum when drag ends
   */
  dragMomentum?: boolean
  
  /**
   * Transition to apply on drag end when momentum is enabled
   */
  dragTransition?: InertiaOptions
  
  /**
   * Allow drag gesture propagation to parent
   */
  dragPropagation?: boolean
  
  /**
   * External controls for drag
   */
  dragControls?: DragControls
}

export interface DragHandlers {
  onDrag?(event: MouseEvent | TouchEvent | PointerEvent, info: PanInfo): void
  onDragStart?(event: MouseEvent | TouchEvent | PointerEvent, info: PanInfo): void
  onDragEnd?(event: MouseEvent | TouchEvent | PointerEvent, info: PanInfo): void
  onDirectionLock?(axis: 'x' | 'y'): void
  whileDrag?: VariantLabels | TargetAndTransition
}
```

### Basic drag

```svelte
<script>
  import { Motion } from 'motion-start'
</script>

<Motion.div
  drag
  dragConstraints={{ left: 0, right: 300, top: 0, bottom: 300 }}
  whileDrag={{ scale: 1.1 }}
>
  Drag me
</Motion.div>
```

### Axis-locked drag

```svelte
<Motion.div drag="x">
  Drag horizontally
</Motion.div>

<Motion.div drag="y">
  Drag vertically
</Motion.div>
```

### Drag with elasticity

```svelte
<Motion.div
  drag
  dragConstraints={{ left: -100, right: 100, top: -100, bottom: 100 }}
  dragElastic={0.7}
>
  Stretchy drag
</Motion.div>
```

<Accordion title="Drag elastic values">
  - `0` = No elasticity, hard constraints
  - `0.5` = Moderate stretch
  - `1` = Very elastic, can drag far beyond constraints
  - `false` = Disable constraints entirely
</Accordion>

### Drag with momentum

```svelte
<Motion.div
  drag
  dragMomentum
  dragTransition={{ 
    power: 0.2,
    timeConstant: 200,
    modifyTarget: target => Math.round(target / 50) * 50
  }}
>
  Flick me
</Motion.div>
```

### Drag with constraints from parent

```svelte
<script>
  import { Motion } from 'motion-start'
  
  let constraintsRef
</script>

<div bind:this={constraintsRef} style="width: 400px; height: 400px; position: relative;">
  <Motion.div
    drag
    dragConstraints={constraintsRef}
  >
    Constrained to parent
  </Motion.div>
</div>
```

## Combining gestures

You can combine multiple gesture handlers on a single element:

```svelte
<script>
  import { Motion } from 'motion-start'
  
  const variants = {
    idle: { scale: 1, rotate: 0 },
    hover: { scale: 1.05 },
    tap: { scale: 0.95 },
    drag: { scale: 1.1, rotate: 5 }
  }
</script>

<Motion.div
  drag
  variants={variants}
  initial="idle"
  whileHover="hover"
  whileTap="tap"
  whileDrag="drag"
  onTap={(e, info) => console.log('Tap:', info.point)}
  onDragEnd={(e, info) => console.log('Drag end velocity:', info.velocity)}
>
  Interactive element
</Motion.div>
```

## Drag controls

Programmatically control drag behavior:

```svelte
<script>
  import { Motion, useDragControls } from 'motion-start'
  
  const dragControls = useDragControls()
  
  function startDrag(event) {
    dragControls.start(event)
  }
</script>

<div>
  <button on:pointerdown={startDrag}>
    Drag handle
  </button>
  
  <Motion.div
    drag="x"
    dragControls={dragControls}
  >
    Drag using the handle
  </Motion.div>
</div>
```

## Best practices

<Accordion title="Set touch-action for touch devices">
  Always set `touch-action: none` (or `touch-action: pan-x`/`pan-y`) on draggable or pannable elements to prevent browser scroll interference on touch devices.
</Accordion>

<Accordion title="Use whileDrag for visual feedback">
  Provide clear visual feedback during drag operations using `whileDrag` animations to indicate the element is being moved.
</Accordion>

<Accordion title="Consider momentum for natural feel">
  Enable `dragMomentum` for a more natural, physics-based feel when users "flick" draggable elements.
</Accordion>

<Accordion title="Combine with layout animations">
  Draggable elements work seamlessly with layout animations. The element will animate to its new position when its layout changes.
</Accordion>

<Accordion title="Test on touch devices">
  Always test gesture interactions on actual touch devices, as mouse and touch behaviors can differ significantly.
</Accordion>
